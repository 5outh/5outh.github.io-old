Intro to the series

Intro (it’s easy in other langs, but it’s effectful)

Randomness is an effectful operation. You need at least two things to produce a random number:

  • An initial “seed” value
  • A pure function that produces a new number from that seed. (“RNG”)

That’ll do, for a single number (How would you produce a second number?). And it’s not really random if your seed is known.

Even still, most programming languages will hide these details from you. Most of the time, you can just call random() and get a random number (typically between 0 and 1). What is really going on, though? In Haskell, we don’t like to do much in IO either. How can we replicate the behavior of random() in our paradigm?

The simplest way is by using the random package, part of System.Random.

We can get by using randomIO and randomRIO to pull from a global RNG:

import System.Random

main :: IO ()
main = do
  -- A random `Double` between 0 and 1
  (randomIO :: IO Double) >>= print

  -- A random `Int` between 1 and 6
  randomRIO (1, 6) >>= print

This is pretty much the interface that most other languages start with. Success! Better would be to separate out any IO as much as possible. Let’s do that, by only using IO to come up with an initial seed.
